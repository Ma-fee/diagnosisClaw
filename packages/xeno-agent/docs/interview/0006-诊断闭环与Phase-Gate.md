# 0006 诊断闭环与Phase-Gate：如何避免“模型会说不会诊”

## 闭环结构
你的诊断流程是分阶段闭环，而不是一次性回答：
1. 信息采集（交互）
2. 诊断规划（可后台研究）
3. 交互执行（逐步检查）
4. 根因确认（置信度 > 90%）
5. 案例沉淀（报告归档）

## 关键机制
### 1) Phase Gate
- 每个阶段需要用户确认才能推进。
- 避免模型“自嗨式”跳步。

### 2) 动态计划更新
- 通过 `update_todo_list` 维护步骤状态。
- 发现根因高置信后可跳过无关步骤，提高效率。

### 3) 安全边界
- 对高风险场景强调人工确认和安全前置。
- 当置信不足或超出用户能力时建议升级处理。

## 面试官问题
### Q1：为什么根因阈值设 90%？
建议回答：
- 这是工程阈值，不是统计学真理。
- 目的是在效率与安全之间折中：低于 90% 继续验证，高于阈值可进入修复建议。

### Q2：为什么 Phase2 可以后台，Phase3 必须交互？
建议回答：
- Phase2 主要是资料检索与计划合成，适合并行计算。
- Phase3 依赖现场测量与人机协作，必须人在回路。

### Q3：如果用户不愿意按步骤走怎么办？
建议回答：
- 系统允许“快速路径”，但会明确告知风险与不确定性。
- 关键安全步骤不可跳过。

## Trade-off
1. 严格分阶段提升正确性，但会牺牲部分响应速度。
2. 允许动态跳步提升效率，但需要严格记录跳步依据。

## 简历表述
- 构建面向工业故障诊断的 Phase-Gate 闭环流程（采集-规划-执行-确认-归档），通过置信度阈值和动态 todo 编排实现诊断效率与安全性的平衡。
