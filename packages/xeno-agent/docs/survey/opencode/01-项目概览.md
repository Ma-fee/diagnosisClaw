# OpenCode 项目概览

## 项目简介

OpenCode 是一个 **100% 开源的 AI 编程助手**，由 Google DeepMind 的高级代理编程团队开发。它提供智能工具使用能力，帮助开发者完成编程任务。

**官网**: https://opencode.ai  
**仓库**: https://github.com/anomalyco/opencode  
**当前版本**: 1.1.25

## 核心目标

1. **100% 开源** - 无供应商锁定
2. **供应商无关** - 支持 Claude、OpenAI、Google、本地模型等多种 LLM
3. **客户端/服务器架构** - 将智能层（服务器）与展示层（CLI、TUI、Web、移动端）分离
4. **专注 TUI** - 由终端爱好者打造，突破终端用户体验边界
5. **内置 LSP 支持** - 一流的语言服务器协议集成
6. **ACP 合规** - 实现 Agent Client Protocol 以实现互操作性

## 与 Claude Code 的差异

OpenCode 与 Claude Code 在功能上非常相似，但有以下关键差异：

- **100% 开源** - 完全透明的代码库
- **供应商无关** - 不绑定特定 AI 提供商
- **内置 LSP 支持** - 开箱即用的代码智能
- **专注 TUI** - 推动终端界面的极限
- **客户端/服务器架构** - 可以在服务器上运行 OpenCode，从远程移动应用控制

## 技术栈

- **运行时**: Bun (快速的 JavaScript 运行时)
- **语言**: TypeScript + ESM 模块
- **验证**: Zod (运行时类型验证)
- **HTTP 服务器**: Hono (轻量级 Web 框架)
- **AI SDK**: Vercel AI SDK
- **前端**: SolidJS + OpenTUI (终端 UI)
- **存储**: 基于文件的 JSON 存储

## 安装方式

```bash
# 快速安装
curl -fsSL https://opencode.ai/install | bash

# 包管理器
npm i -g opencode-ai@latest        # npm/bun/pnpm/yarn
brew install anomalyco/tap/opencode # macOS/Linux (推荐)
choco install opencode              # Windows
paru -S opencode-bin                # Arch Linux
```

## 内置代理

OpenCode 包含可通过 `Tab` 键切换的两个主要代理：

1. **build** - 默认代理，具有完全访问权限，用于开发工作
2. **plan** - 只读代理，用于分析和代码探索
   - 默认拒绝文件编辑
   - 执行 bash 命令前需要权限
   - 适合探索不熟悉的代码库或规划变更

还包括用于复杂搜索和多步骤任务的 **general** 子代理，可通过消息中的 `@general` 调用。

## 开发命令

```bash
# 安装依赖
bun install

# 运行开发模式
bun dev

# 类型检查
bun run typecheck

# 运行测试
bun test

# 构建
bun run build
```

## 主要特性

1. **多代理系统** - 支持主代理和子代理
2. **丰富的工具集** - bash、文件操作、LSP、搜索等
3. **权限系统** - 细粒度的工具访问控制
4. **会话管理** - 支持会话创建、分叉、压缩
5. **插件系统** - 通过 npm 包或本地文件扩展
6. **事件驱动架构** - 中央事件总线用于实时更新
7. **上下文管理** - 自动压缩和修剪策略
8. **客户端/服务器** - 支持多客户端连接

## 配置文件位置

配置合并优先级（从低到高）：

1. 远程配置（来自认证提供商的 `.well-known/opencode`）
2. 全局配置（`~/.config/opencode/config.json`）
3. 自定义配置路径（通过标志）
4. 项目配置（项目中的 `opencode.jsonc`）
5. 目录配置（祖先目录中的 `.opencode/opencode.jsonc`）
6. 内联配置内容（通过标志）

## 数据存储位置

```
~/.local/state/opencode/storage/
├── session/         # 会话元数据
├── message/         # 消息

├── project/         # 项目信息
├── session_diff/    # 会话差异
└── migration/       # 迁移信息
```
