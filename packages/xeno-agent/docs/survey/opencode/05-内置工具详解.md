# 内置工具详解

## Bash Tool

**文件位置**: `src/tool/bash.ts`

Bash 工具是 OpenCode 最强大的功能之一，但也最危险。因此它的设计包含了多层安全机制。

### 1. 命令解析与安全检查

使用 `tree-sitter-bash` 对命令进行 AST 解析，识别：

- 简单的命令名 (Command Name)
- 参数 (Arguments)
- 重定向 (Redirection)

**安全策略**:

- 检查是否存在高风险命令 (`rm`, `mv`, `cp`, `chmod` 等)
- 如果检测到高风险命令，且权限未显式允许，则触发 `ask`
- 解析所有路径参数，确保不会越界访问 (External Directory Check)

### 2. 执行机制

- 使用 `Bun.spawn` 执行命令
- 支持 `timeout` (默认 2 分钟)
- 支持 `workdir` (避免使用 `cd && cmd` 模式)
- 实时捕获 `stdout` 和 `stderr`
- 自动截断输出 (超过 30KB 或 2000 行)

### 3. Prompt 设计

Prompt (`src/tool/bash.txt`) 强调：

- 必须引用包含空格的路径 (`"path with spaces"`)
- 必须先验证目录是否存在 (`ls`) 再创建
- 避免使用交互式命令 (`git add -i`)
- 总是并行执行独立的命令

## Edit Tool

**文件位置**: `src/tool/edit.ts`

Edit 工具是代码修改的核心，采用"查找-替换"模式。

### 1. 替换策略 (Replacer Strategies)

Edit 工具不仅仅是简单的字符串替换，它实现了一个智能的策略链：

1. **SimpleReplacer**: 精确字符串匹配 (最快，最准确)
2. **LineTrimmedReplacer**: 忽略行首尾空格的匹配
3. **BlockAnchorReplacer**: 基于首尾行锚点的模糊匹配 (Levenshtein 距离)
4. **ContextAwareReplacer**: 基于上下文的模糊匹配

如果一种策略失败，会自动尝试下一种。这大大提高了 LLM 生成代码略有偏差时的成功率。

### 2. LSP 集成

编辑完成后，会自动触发 LSP 检查：

1. 等待文件变更事件
2. 请求 LSP Diagnostics
3. 如果发现新错误，将错误信息附加到工具输出中

这形成了一个紧密的反馈循环：`Edit -> Error -> LLM Fix -> Edit`。

### 3. 安全检查

- 禁止空 `oldString` (防止意外覆盖)
- 检查文件是否为二进制文件
- 生成 Diff 供用户确认

## Task Tool

**文件位置**: `src/tool/task.ts`

Task 工具用于分派任务给子 Agent。

### 1. 递归 Session

每次调用 `task` 都会创建一个新的子 Session：

```typescript
const session = await Session.create({
  agent: "general", // 或指定 agent
  parentSessionID: currentSession.id,
});
```

### 2. 工具屏蔽

为了防止递归死循环，子 Session 会自动禁用某些工具：

- `todowrite` / `todoread`: 子任务不应修改主任务列表
- `task`: (可选) 防止无限递归调用

### 3. 结果传递

子 Session 的最后一条消息内容作为工具的输出返回给父 Session。

## 其他关键工具

### Grep Tool

- 使用 `ripgrep` (rg) 的高性能搜索
- 支持正则、glob 过滤
- 限制输出大小，防止淹没上下文

### Glob Tool

- 快速查找文件路径
- 按修改时间排序，优先展示最近修改的文件

### WebFetch Tool

- 使用 `fetch` 获取网页
- 自动将 HTML 转换为 Markdown (使用 `turndown`)
- 提取元数据 (Title, Description)

## 工具注册表 (Registry)

所有工具通过 `src/tool/index.ts` 集中导出，并在 Session 启动时加载。

```typescript
export const tools = [
  BashTool,
  EditTool,
  ReadTool,
  // ...
];
```

这种集中式管理使得添加新工具或禁用特定工具变得非常容易。
