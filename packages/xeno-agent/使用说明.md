# Xeno-Agent 使用说明

## 当前状态

✅ 系统框架已完整实现，包括：
- 核心 State, Flow, Signals 模块
- Agent 构建和 YAML 加载
- HITL 支持和 approval 机制
- CLI 入口点
- 完整的 pytest 测试套件
- LLM 集成（litellm + 环境变量配置）

⚠️ 需要配置正确的 LLM API 信息

## 安装依赖

```bash
cd packages/xeno-agent
uv sync
```

## 配置

### 方法 1: 使用 .env 文件

```bash
cp .env.example .env
# 编辑 .env 文件，填入正确的 API 配置
```

### 方法 2: 使用 direnv

```bash
cp .envrc.example .envrc
direnv allow
```

### 方法 3: 使用命令行参数

每次运行时指定配置：
```bash
uv run xeno-agent test --api-base "YOUR_API_BASE" --model "openai/YOUR_MODEL"
```

## 运行测试

### 测试 LLM 连接

```bash
# 使用环境变量配置
uv run xeno-agent test

# 或使用命令行参数
uv run xeno-agent test --api-base "YOUR_API_BASE" --model "openai/YOUR_MODEL"
```

### 运行模拟

```bash
# 使用默认配置（需要先设置环境变量）
uv run xeno-agent run "创建一个简单的 Python 函数来计算斐波那契数列"

# 指定 initial mode
uv run xeno-agent run "代码审查任务" --mode qa

# 使用自动审批模式（跳过工具确认）
uv run xeno-agent run "构建一个网络爬虫" --auto-approve

# 使用禁用 HITL 模式 (全自动, 无需人工确认)
uv run xeno-agent run "构建一个网络爬虫" --disable-hitl
```

## 运行单元测试

```bash
# 运行所有测试
uv run pytest

# 运行测试并显示覆盖率
uv run pytest --cov=xeno_agent

# 运行特定测试文件
uv run pytest tests/test_state.py
```

## 注意事项

### LLM 模型配置

LiteLLM 要求模型名格式为：`provider/model_name`

常见 provider：
- `openai/` - OpenAI 格式 API（包括兼容 OpenAI 格式的自定义 API）
- `anthropic/` - Anthropic Claude
- `google/` - Google PaLM
- `huggingface/` - HuggingFace

**当前配置示例**：
```bash
# 对于自定义 OpenAI 兼容 API
export XENO_LLM_LLM_MODEL="openai/glm-4.7"
export XENO_LLM_API_BASE="http://api.ai.rootcloud.info/v1"
```

### Agent 模式

内置的 Agent 模式（可在 `src/xeno_agent/simulation/agents/` 目录查看）：
- `pm` - Project Manager
- `developer` - Developer
- `qa` - QA Engineer

创建自定义 Agent：
1. 在 `src/xeno_agent/simulation/agents/` 目录创建 YAML 文件
2. 格式示例：
```yaml
role: "Your Agent Name"
goal: "Agent's goal"
backstory: "Detailed backstory"
skills:
  - switch_mode
  - new_task
  - attempt_completion
```

### 常见问题

1. **连接测试失败**: 检查 API_BASE 和 MODEL 是否正确
2. **模型找不到**: 确认 API endpoint 支持该模型
3. **模块导入错误**: 运行 `uv sync` 安装依赖
4. **LSP 报错**: 这是正常的，因为 CrewAI 包在编辑器中可能不可见，运行时不会影响

## 架构说明

系统基于 CrewAI Flow 模式，使用栈式状态机管理任务流转：

- **GOTO** (`switch_mode`): 切换当前 Agent 模式
- **GOSUB** (`new_task`): 创建新的子任务（压栈）
- **RETURN** (`attempt_completion`): 完成当前任务（出栈）
- **HITL** (`@human_feedback`): 人机交互/审批 (Flow 级集成)
